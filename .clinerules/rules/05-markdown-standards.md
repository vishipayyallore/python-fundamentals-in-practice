# Markdown Standards

Canonical detail: `.cursor/rules/04_markdown-standards.mdc`.

- Keep one H1 per document and use heading levels in order.
- Use fenced code blocks with language identifiers.
- Prefer relative links that work from the file location.
- Validate file references before committing, especially `docs/sessions/L{level}/S{session}.md` and `src/L{level}/S{session}/`.
- Avoid corrupted characters and preserve the existing educational formatting style.
