# Session 9: Dictionaries & Basic Testing

**Duration:** 30 minutes  
**Type:** 📚 Knowledge  
**Level:** Noob → Nerd

---

## 🎯 Session mission: label your data with meaningful keys

Lists order items by position; **dictionaries** map **keys** to **values** — perfect for records like a student, a product, or a user profile.

### What you'll learn

- Create dicts with `{key: value}` literals.
- Read, update, and delete entries.
- Loop with `.keys()`, `.values()`, and `.items()`.
- Use **`.get()`** when a key might be missing.
- Write small **`assert`** checks to verify your own logic while learning.

## 🧰 Troubleshooting

1. **KeyError**  
   The key is not in the dict — use `.get(key, default)` or check `if key in record` first.

2. **AssertionError**  
   An `assert` failed — your assumption was wrong; fix the code or the expected value.

3. **Mutable values shared by surprise**  
   Putting a list inside a dict is fine — just remember the **same list object** can be changed in place.
