# Session 7: Basic Debugging & Built-in Functions

**Duration:** 30 minutes  
**Type:** 📚 Knowledge  
**Level:** Noob → Nerd

---

## 🎯 Session mission: errors are messages, not mysteries

Session 6 gave you loops that run many times — one small bug can repeat forever. This session teaches you to **read what Python tells you** and to **peek inside running code** with `print()` before you change logic.

### What you'll learn

- Tell **syntax errors** (file won't start) from **runtime errors** (program starts, then fails).
- Recognize **NameError**, **TypeError**, **ZeroDivisionError**, and **ValueError** from their messages.
- Use **`print()` checkpoints** to see values at each step.
- Apply everyday **built-ins**: `len()`, `max()`, `min()`, `sum()`, `abs()`, `round()`.
- Revisit **PEP 8** style so debug output stays readable.

## 🧰 Troubleshooting

1. **"I only see the last line of the error"**  
   Scroll up — the **traceback** lists file names and line numbers from bottom to top.

2. **SyntaxError before the program runs**  
   Check missing `:`, quotes, or indentation — Python never reached your logic.

3. **Too many DEBUG prints**  
   Comment them out once you understand the flow — leave one or two at decision points.
