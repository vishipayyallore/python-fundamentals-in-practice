## 🔁 Part 1: `for` loops (fixed repetition)

Use `for` when you know the repetition shape upfront.

- `range(5)` means 5 iterations: `0,1,2,3,4`
- `range(1, 6)` means `1..5`
- `range(10, 0, -1)` counts down

Why this matters: beginners often type the same print/input logic many times.  
A loop keeps code shorter, clearer, and easier to update later.

## ⏳ Part 2: `while` loops (condition repetition)

Use `while` when repetition depends on a condition.

Core pattern:

1. Start value (setup)
2. Condition check
3. Update value inside the loop

Why this matters: if you forget the update, the condition never changes and you get an infinite loop.

## 🧭 Part 3: loop control keywords

- `break` → stop the loop immediately
- `continue` → skip current iteration and move to next
- `pass` → placeholder that does nothing (use sparingly)

Reasoning tip: choose the one that matches intent.

- Need to exit fully? `break`
- Need to ignore one bad case? `continue`
- Need a temporary structural placeholder? `pass`

## 🧰 Troubleshooting (common mistakes)

1. **Infinite `while` loop**
   - Cause: loop variable never changes.
   - Fix: update the variable each iteration (`count -= 1`, etc.).

2. **Off-by-one with `range()`**
   - Cause: expecting `range(1, 5)` to include `5`.
   - Fix: remember the stop value is excluded.

3. **Wrong indentation**
   - Cause: lines intended for loop body are outside it.
   - Fix: align all loop-body lines at same indentation level.

4. **`break`/`continue` confusion**
   - Fix memory trick: `break = end`, `continue = skip`.
