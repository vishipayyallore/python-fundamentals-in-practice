# Session 3: Operators & Expressions

**Duration:** 30 minutes (run code and **predict**; long operator tables = optional reading.)  
**Type:** 📚 Knowledge  
**Level:** Noob → Nerd

---

## The feeling you want: **“I can compute and evaluate things using data.”**

This is the shift from **syntax** to **logic thinking**: you are not listing operators — you are **acting on values** (arithmetic) and **asking true/false questions** (comparisons). That **True/False** is exactly what **Session 4** uses for **if/else**.

> **Mental model 1:** *Operators let you **act** on data (add, compare, …).*  
> **Mental model 2:** *Expressions **evaluate to a value** — e.g. `5 + 3` → `8`, `10 > 5` → `True`.*  
> **Mental model 3 (bridge to decisions):** **Comparison** expressions produce **`True` or `False`** — that is the foundation of **branching** next session.

### Hook (run in REPL or a tiny script)

```python
age = 25
print(age >= 18)
```

**Ask:** *Is this `True` or `False`?* Then: *This is how programs start **testing** the world before they **decide** — you’ll add `if` in Session 4.*

## Why operators matter (after Session 2)

In Session 2 you **stored** data and **knew its type**. You had not yet **done much** with that data in one place.

**Operators let you:**

- **Calculate** (arithmetic)
- **Compare** (get `True`/`False`)
- **Build expressions** you can print, store, and later use in conditions

This is where programs start to feel **useful**, not just talkative.

## Where people stumble (say it out loud)

| Confusion | Teaching line |
| --------- | ---------------- |
| **`10 / 2` → `5.0`** | In Python 3, **`/` is true division** — result is a **float**, even when it looks “even.” |
| **`=` vs `==`** | **`=`** stores; **`==`** compares. The #1 mix-up for beginners. |
| **`2 + 3 * 4` → `14` not `20`** | `*` before `+` — **precedence**; use **`()`** to force order. |

## Concepts (compressed)

### Arithmetic (high level)

- `+` `-` `*` — usual math.
- **`/`** — always float in Python 3.
- **`//`** — floor division (drops toward “floor” for negatives; learn by examples in `01_arithmetic.py`).
- **`%`** — remainder; useful for even/odd, wrapping.
- `**` — power; `2**3**2` follows Python’s **right-associative** rule for `**` (see script output).

### Comparisons (high level)

- `==` `!=` `<` `>` `<=` `>=` — all give **bool**.
- **Chaining:** `18 <= age <= 65` reads like math; still one bool.

### Precedence

- When unsure, **parentheses** — no prize for “doing it in your head” in production.

### Assignment “shortcuts” (`+=`, …)

- Useful when you want to update the same variable with a new result.
- Keep them light in a first pass: learners should understand `x = x + 5` first, then recognize that `x += 5` means the same update more compactly.

```python
score = 15
score += 10   # same idea as: score = score + 10
score *= 2    # same idea as: score = score * 2
```

Use them as a readability shortcut, not as something to memorize in isolation.

## Quick reference

```python
# Arithmetic
10 / 3    # 3.333... float

# Compare → bool

x == y

# Parens
(2 + 3) * 4  # 20
```

## Troubleshooting (short)

- **Surprise `5.0`:** you used `/` — use `//` for “integer division feel,” or `int()` with care.  
- **`SyntaxError` with `=` in an expression:** you used assignment where only **compare** or **pass value** is allowed.  
- **Wrong math order:** add **`()`** around what must happen first.
