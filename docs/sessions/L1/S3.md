## The feeling you want: **‚ÄúI can compute and evaluate things using data.‚Äù**

This is the shift from **syntax** to **logic thinking**: you are not listing operators ‚Äî you are **acting on values** (arithmetic) and **asking true/false questions** (comparisons). That **True/False** is exactly what **Session 4** uses for **if/else**.

> **Mental model 1:** *Operators let you **act** on data (add, compare, ‚Ä¶).*  
> **Mental model 2:** *Expressions **evaluate to a value** ‚Äî e.g. `5 + 3` ‚Üí `8`, `10 > 5` ‚Üí `True`.*  
> **Mental model 3 (bridge to decisions):** **Comparison** expressions produce **`True` or `False`** ‚Äî that is the foundation of **branching** next session.

### Hook (run in REPL or a tiny script)

```python
age = 25
print(age >= 18)
```

**Ask:** *Is this `True` or `False`?* Then: *This is how programs start **testing** the world before they **decide** ‚Äî you‚Äôll add `if` in Session 4.*

## Why operators matter (after Session 2)

In Session 2 you **stored** data and **knew its type**. You had not yet **done much** with that data in one place.

**Operators let you:**

- **Calculate** (arithmetic)
- **Compare** (get `True`/`False`)
- **Build expressions** you can print, store, and later use in conditions

This is where programs start to feel **useful**, not just talkative.

## Where people stumble (say it out loud)

| Confusion | Teaching line |
| --------- | ---------------- |
| **`10 / 2` ‚Üí `5.0`** | In Python 3, **`/` is true division** ‚Äî result is a **float**, even when it looks ‚Äúeven.‚Äù |
| **`=` vs `==`** | **`=`** stores; **`==`** compares. The #1 mix-up for beginners. |
| **`2 + 3 * 4` ‚Üí `14` not `20`** | `*` before `+` ‚Äî **precedence**; use **`()`** to force order. |

## Concepts (compressed)

### Arithmetic (high level)

- `+` `-` `*` ‚Äî usual math.
- **`/`** ‚Äî always float in Python 3.
- **`//`** ‚Äî floor division (drops toward ‚Äúfloor‚Äù for negatives; learn by examples in `01_arithmetic.py`).
- **`%`** ‚Äî remainder; useful for even/odd, wrapping.
- `**` ‚Äî power; `2**3**2` follows Python‚Äôs **right-associative** rule for `**` (see script output).

### Comparisons (high level)

- `==` `!=` `<` `>` `<=` `>=` ‚Äî all give **bool**.
- **Chaining:** `18 <= age <= 65` reads like math; still one bool.

### Precedence

- When unsure, **parentheses** ‚Äî no prize for ‚Äúdoing it in your head‚Äù in production.

### Assignment ‚Äúshortcuts‚Äù (`+=`, ‚Ä¶)

- Useful; **not** the center of a 30-minute first pass ‚Äî keep in exercises or a later pass if you run long.

## Troubleshooting (short)

- **Surprise `5.0`:** you used `/` ‚Äî use `//` for ‚Äúinteger division feel,‚Äù or `int()` with care.  
- **`SyntaxError` with `=` in an expression:** you used assignment where only **compare** or **pass value** is allowed.  
- **Wrong math order:** add **`()`** around what must happen first.
