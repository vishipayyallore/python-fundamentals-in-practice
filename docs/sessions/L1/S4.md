## 🪝 Opening hook: when “always the same” is not enough

👉 Until now, programs were predictable. Today, they become dynamic.

**So far, most programs are one straight path:** they run the same way every time. That’s great for learning mechanics—it’s *predictable*.

```python
print("You are an adult")
```

Ask (out loud, or to yourself in the doc): **What if the user is 10?** A single `print` cannot *decide*—it only announces.

**Introduce the first branch:**

```python
age = 10

if age >= 18:
    print("Adult")
else:
    print("Minor")
```

That is the “aha” for this level: **the program can follow different lines depending on a condition.** Everything else in this session unpacks *that* idea: shape (indentation), more branches (`elif`), combining tests (`and` / `or` / `not`), and one bolt-on “extra power” (`random`).

## 📐 **Part 1: Python Indentation (keep it sharp)**

**One sentence:** in Python, **indentation is structure**—it plays the same role that `{}` does in C-like languages, but *you* keep it visible so “what belongs together” is obvious from shape.

**One deliberate mistake—run and read the error:**

```python
if True:
print("Hello")
```

Python expects an **indented block** under `if`—the next line has to be indented. Fix:

```python
if True:
    print("Hello")
```

**The usual good habit:** one level = **4 spaces** (pep 8), same style everywhere; keep **one** style, don’t mix tabs and spaces. Editors like VS Code usually indent after a line ending with `:`.

### ✅ **Progress check: indentation (minimal)**

- [ ] After a colon, the “body” is **indented**
- [ ] **Same** number of spaces per level (4 is standard for this course)
- [ ] A line that is not under the `if` must **not** be half-indented by accident

## 🤔 **Part 2: Conditionals—how programs make decisions**

Conditional statements are **the branch points**: depending on a condition (usually something that evaluates to `True` or `False`), the program **runs one block and skips all remaining branches** (for a single chain, at most one branch in an `if` / `elif` / `else` chain).

**Teach in this order** (one idea per step):

### Step 1: only `if`

```python
score = 95

if score >= 90:
    print("A")
```

**Meaning:** if the condition is false, the indented part simply does not run. No `else` yet.

### Step 2: `if` / `else` (two outcomes)

```python
score = 40

if score >= 50:
    print("Pass")
else:
    print("Fail")
```

**Meaning:** exactly one of the two runs — never both.

### Step 3: `if` / `elif` / `else` (more than two buckets)

You can chain as many `elif` branches as you need — Python checks each one in order and runs the **first** branch whose condition is true, then skips the rest.

```python
score = 82

if score >= 90:
    print("A")
elif score >= 80:
    print("B")
elif score >= 70:
    print("C")
elif score >= 60:
    print("D")
else:
    print("F")
```

**How Python decides:** it walks **from top to bottom** and uses the first branch whose condition is true—then it **stops**. That is why the order of `if` / `elif` **matters** (put the stricter / higher band first when splitting ranges).

👉 If conditions overlap, order becomes logic.

👉 Only one branch runs in an `if` / `elif` / `else` chain.

**Wrong order — the "aha" moment:**

```python
# ⚠️ Wrong order example (will break logic)
score = 95

if score >= 75:
    print("B")
elif score >= 90:
    print("A")   # ❌ Never reached
```

👉 Python stops at the first True condition — `score >= 75` is true for 95, so `"A"` is never printed.

👉 It does NOT check remaining conditions after that.

### **Nested Conditionals**

You can put `if` statements inside other `if` statements:

```python
has_ticket = True
age = 15

if has_ticket:
    if age >= 18:
        print("Welcome to the adult section!")
    else:
        print("Welcome to the family section!")
else:
    print("Please buy a ticket first.")
```

- [ ] Use `if-else` for two choices
- [ ] Chain conditions with `elif`
- [ ] Understand that only one block executes

## 🔗 **Part 3: Boolean logic (where people mix ideas—keep it concrete)**

You often need **more than one fact** to decide. Use **`and`**, **`or`**, and **`not`** in plain language first, then in code. Prefer **short, real** situations: **driving**, **open days**, **discounts**, **login**—not abstract `p` and `q` unless you are debugging.

**Combining conditions — three operators:**

Sometimes you need to check multiple conditions together. Python provides three Boolean operators:

| Operator | Description                      | Example                     |
| -------- | -------------------------------- | --------------------------- |
| `and`    | Both conditions must be True     | `age >= 18 and has_id`      |
| `or`     | At least one condition is True   | `is_student or is_senior`   |
| `not`    | Reverses the Boolean value       | `not is_banned`             |

### **The `and` Operator**

Both conditions must be `True`:

```python
age = 25
has_license = True

if age >= 18 and has_license:
    print("You can drive!")
else:
    print("You cannot drive.")

# Truth table for 'and':
# True and True   = True
# True and False  = False
# False and True  = False
# False and False = False
```

### **The `or` Operator**

At least one condition must be `True`:

```python
is_weekend = True
is_holiday = False

if is_weekend or is_holiday:
    print("You can sleep in! 😴")
else:
    print("Time to wake up early! ⏰")

# Truth table for 'or':
# True or True   = True
# True or False  = True
# False or True  = True
# False or False = False
```

### **The `not` Operator**

Reverses `True` to `False` and vice versa:

```python
is_raining = False

if not is_raining:
    print("Let's go outside! 🌞")
else:
    print("Better stay inside. 🌧️")

# not True  = False
# not False = True
```

### **Combining Multiple Operators**

```python
age = 25
is_student = True
has_coupon = False

# Complex condition
if (age < 30 and is_student) or has_coupon:
    print("You get a discount!")
else:
    print("Regular price.")
```

**Use parentheses to make complex conditions clear!**

### **Practical Example: Login Validation**

```python
username = input("Enter username: ")
password = input("Enter password: ")

correct_user = "admin"
correct_pass = "secret123"

if username == correct_user and password == correct_pass:
    print("✅ Login successful!")
elif username != correct_user:
    print("❌ Unknown user")
else:
    print("❌ Wrong password")
```

### ✅ **Progress Check: Boolean Logic**

Make sure you understand:

- [ ] `and` requires BOTH conditions to be True
- [ ] `or` requires AT LEAST ONE condition to be True
- [ ] `not` reverses the Boolean value
- [ ] Use parentheses for complex conditions

## 📦 **Part 4: Modules—keep the first exposure light**

👉 So far, decisions use our data. Now we add randomness to make programs less predictable.

**Idea in one line:** the core language is **small**; much of the “extra power” lives in **modules** (libraries of names you **import** when you need them).

- **Built-ins** (no import): `print`, `input`, `len`, `type`, …
- **Module** (import first): a named bundle of extra functions, e.g. `import random` then `random.randint(...)`

**This session’s scope:** you only need to **name** that pattern and use **one** module well: `random` for a fair roll or secret number.

```python
import random

print(random.randint(1, 10))
```

**Pedagogy line you can use:** *Modules are how we bolt on ability without memorizing a giant language on day one—import what the lesson needs.*

**Optional later (not required for L1 S4):** `math`, `random.choice`, `random.shuffle`, seeds, etc. The practice file `03_number_guessing_game.py` in the repo may use a bit more than the minimal snippet; that is fine for exploration—**in the room, don’t open the whole standard library**; keep attention on **one decision** + **one import**.

### ✅ **Progress check: modules (minimal for today)**

- [ ] Know that **`import name`** is how you load a module
- [ ] Know that **`name.function(...)`** calls a function from that module
- [ ] Can use **`random.randint(a, b)`** for an inclusive integer range

## 🎲 **Part 5: High-impact demo—number guessing (one screen)**

Walk this **slowly**—it is the “everything snapped together” moment: **operator**, **condition**, **branch**, **extra power** from `random`, **`input`**.

👉 This program combines input + comparison + decision.

```python
import random

secret = random.randint(1, 10)
```

## ⚠️ **Part 6: Call these mistakes out (or they will happen)**

Say them out loud in the room (or mark them in your notes):

| Pitfall | What goes wrong | Quick fix mental model |
| ------- | ----------------- | ---------------------- |
| `=` vs `==` in a condition | `if x = 5` is a **syntax** problem—`=` is assignment | In `if`, you almost always want **compare**: `==` |
| Missing `:` | `if` line does not end the header | The line with `if` / `elif` / `else` needs **`:`** before the block |
| Wrong / missing indent | `IndentationError` | The block under `if` is **indented**; one level = 4 spaces in this course |
| `input` without `int` when you need a number | You compare **strings** to **numbers** by accident | `age = int(input("Age: "))` when the plan is numeric |

## 📝 **Key takeaways**

### **What this session is really about**

- ✅ **Decisions, not more syntax for its own sake**—conditionals are how a program *chooses*
- ✅ **Shape matters**—indentation = block structure; colons end the header of `if` / `elif` / `else`
- ✅ **Order matters** in `if` / `elif` / `else`—**first true branch wins**; the rest of the chain is skipped
- ✅ **Booleans combine real tests** with `and`, `or`, `not`
- ✅ **One module, one job (today):** `import random` → `randint` for a secret or roll
- ✅ The **number-guessing** idea ties it together; **loops** come when you need **many** guesses

### **Quick Reference:**

```python
# Conditionals
if condition:
    # code
elif other_condition:
    # code
else:
    # code

# Boolean operators
both_true = cond1 and cond2
either_true = cond1 or cond2
opposite = not condition

# Random module
import random
random.randint(1, 10)    # Random int
```

## 🔧 **Troubleshooting**

If something fails, re-check **Part 6: call these mistakes out** first (`=` vs `==`, colons, indentation, `int(input())`). Short reminders:

- **`IndentationError`:** body of `if` is not indented, or you mixed tabs and spaces.  
- **`:` missing:** the line with `if` / `elif` / `else` must end with **`:`** before the block.  
- **`SyntaxError` on `if x = 5`:** you used **assignment** where you need **compare** `==`.
