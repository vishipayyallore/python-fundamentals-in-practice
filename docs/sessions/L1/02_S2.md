# Session 2: Variables & Data Types

**Duration:** 30 minutes (prioritize **variables → kinds of data → input & conversion**; long type theory is optional reading.)  
**Type:** 📚 Knowledge  
**Level:** Noob → Nerd

---

## The feeling you want: **“I can store data and know what kind of data I’m using.”**

Not only “we learned int and str” — but: **I put something in a name**, I **inspected** it with `type()`, and I **fix** the “`10` + `5` is not `15`” surprise when the data is text.

> **Mental model 1:** *Variables **store** values. **Types** describe what **kind** of value it is.*  
> **Mental model 2:** *Python **remembers** the value; **you** must know whether it is text, a whole number, a decimal, or true/false for the next step.*  
> **Mental model 3 (most important in practice):** `input()` **always** returns **text (str)**. If you want **numbers**, **convert** (`int`, `float`, …).

## Why variables matter (connect to Session 1)

In Session 1 you **took input** and **printed** output. You did **not** yet **hold** that data under a name for multiple steps in a clear way (beyond what `input` returns on one line).

**Variables fix that:** they let a program **remember**, **reuse**, and **change** information under a **name** you choose.

```python
name = input("Enter your name: ")
print(name)
print(name)
```

**In the room, ask:** *Why does it print twice?* *Where is the value stored?* (In memory, under the name `name` — you’ll **reuse** the same name again until you reassign it.)

The practice file `01_variables.py` starts with this pattern, then shows reuse with a literal and larger examples.

## Where people struggle (call these out)

| Confusion | Quick fix in teaching |
| --------- | ------------------------ |
| **`"25"` vs `25`** | Show `type` side by side. **Quotes = text** even if characters are digits. |
| **`input()` is always a string** | Re-run Session 1’s observation with intent: **convert** before `+` for math. |
| **`int("hello")` crashes** | Only valid number text can become `int` / `float`—errors come later. |
| **String `+` vs number `+`** | `"10" + "5"` → `"105"`. `10 + 5` → `15`. **Ask → run → wince** — that moment is the lesson. |

## Concepts (compressed for class)

### Variables and names

- **Assignment:** `name = value` — the **name** (left) **refers to** the **value** (right).
- **Rules:** no spaces in names, don’t start with a digit, avoid shadowing `print`/`if`, prefer `snake_case` and **readable** words.

### Four types you will see everywhere

| Type | Examples | One-line use |
| ---- | -------- | -------------- |
| `int` | `25`, `-1`, `0` | Counts, indexes, whole numbers |
| `float` | `3.14`, `19.99` | Measurements, decimals |
| `str` | `"hi"`, `'25'` | Text, including **digit-looking text** from `input` |
| `bool` | `True`, `False` | Flags and conditions (later sessions lean hard on this) |

### Dynamic typing (short)

You don’t **declare** `int x` in Python. The **value** you assign tells you the type; the **same** name can later refer to a **new** value (and maybe a **new** type). For beginners: **one step at a time, check** with `type()` when unsure.

### Checking types

- **`type(x)`** — your default “what is this?” tool in early levels.
- **`isinstance(x, int)`** — “is this *specifically* an int?” Useful in real code; in **30 minutes**, show **one** example (see `02_data_types.py`); do **not** run a long `isinstance` menu in lecture.

### Conversion (center of the session)

- `int("25")` → `25`  
- `float("3.5")` → `3.5`  
- `str(25)` → `"25"`  
- `bool(...)` has subtleties — **deeper truth tables** = homework or a later level, not the whole period.

**`03_type_conversion.py`** leads with: **two strings that look like numbers** → **`+` glues text**; **`int` then** → real addition. **Then** the line tying **`input()` to `str`**. That’s the path to **Session 3** (operate on real numbers and compare).

## Troubleshooting (short)

- **`TypeError: ... str and int`:** you mixed types without `str()` or an f-string.  
- **`ValueError: invalid literal for int()`:** the string wasn’t a **plain** number (spaces, text, or decimal when you used `int`).  
- **Wrong `+`:** re-check: both operands **str**? both **int/float**?
