## 🔁 Part 1: `for` loops (fixed repetition)

Use `for` when you know the repetition shape upfront.

- `range(5)` means 5 iterations: `0,1,2,3,4`
- `range(1, 6)` means `1..5`
- `range(10, 0, -1)` counts down

Why this matters: beginners often type the same print/input logic many times.  
A loop keeps code shorter, clearer, and easier to update later.

## ⏳ Part 2: `while` loops (condition repetition)

Use `while` when repetition depends on a condition.

Core pattern:

1. Start value (setup)
2. Condition check
3. Update value inside the loop

Why this matters: if you forget the update, the condition never changes and you get an infinite loop.

## 🧭 Part 3: loop control keywords

- `break` → stop the loop immediately
- `continue` → skip current iteration and move to next
- `pass` → placeholder that does nothing (use sparingly)

Reasoning tip: choose the one that matches intent.

- Need to exit fully? `break`
- Need to ignore one bad case? `continue`
- Need a temporary structural placeholder? `pass`

### Raw values to variables (walkthrough)

Start with literal values — Python evaluates them directly. Then see why variables avoid rewriting the same value:

```python
# Raw literals
7        # int
5.9      # float
"hello"  # str
True     # bool

# Arithmetic on those values
7 * 7       # 49 — stays int
5.2 + 6.8   # 12.0 — stays float

# Use type() to inspect
type(7)        # <class 'int'>
type("hello")  # <class 'str'>
```

Then assign to names to avoid repetition:

```python
a = "november"
print(a)  # november — reuse without retyping

a = "october"  # reassignment
print(a)  # october — the new value
```

### Chained and multiple-target assignment

Compact assignment forms:

**Chained assignment** — give the same value to several names:

```python
north = south = central = east = 2
print(north, south, central, east)  # 2 2 2 2
```

**Multiple-target assignment (tuple unpacking)** — give different values in one statement:

```python
city, hub = "Hyderabad", "Pune"
print(city, hub)  # Hyderabad Pune

# Swap two variables without a temporary
warehouse_a, warehouse_b = warehouse_b, warehouse_a
```

**Note:** The number of names on the left must match the number of values on the right, or Python raises a `ValueError`.

### Conversion limits — what you lose, and what fails

**`int(float)` truncates** (does NOT round):

```python
print(int(5.8))   # 5, not 6
print(int(-3.9))  # -3, not -4
```

**`float(bool)` follows the bool-as-int rule:**

```python
print(float(True))   # 1.0
print(float(False))  # 0.0
```

**`int()` on a non-numeric string raises `ValueError`:**

```python
try:
    result = int("september")
except ValueError as err:
    print(f"ValueError: {err}")  # ValueError: invalid literal for int() with base 10: 'september'

# A numeric string works fine
converted = int("87")
print(converted)  # 87
```

**Note:** `int()` succeeds only when the string contains digits (optionally preceded by `+` or `-`).

## 🧰 Troubleshooting (loops)

1. **Infinite `while` loop**
   - Cause: loop variable never changes.
   - Fix: update the variable each iteration (`count -= 1`, etc.).

2. **Off-by-one with `range()`**
   - Cause: expecting `range(1, 5)` to include `5`.
   - Fix: remember the stop value is excluded.

3. **Wrong indentation**
   - Cause: lines intended for loop body are outside it.
   - Fix: align all loop-body lines at same indentation level.

4. **`break`/`continue` confusion**
   - Fix memory trick: `break = end`, `continue = skip`.

## 🧰 Troubleshooting (Session 2 extensions)

1. **`ValueError: not enough values to unpack`**
   - Cause: Number of names ≠ number of values in multi-assignment.
   - Fix: Count carefully — `a, b = 1, 2` (2 names, 2 values); `a, b = (1, 2, 3)` fails (3 values, 2 names).

2. **`ValueError: invalid literal for int()`**
   - Cause: String contains non-numeric characters.
   - Fix: Validate the string is numeric before conversion (e.g., `"87"` → `int("87")` OK; `"87x"` → fails).

3. **`int()` on float truncates instead of rounding**
   - Cause: `int()` always moves toward zero, not rounding.
   - Fix: Use `round()` if you need conventional rounding.

4. **Type mismatch on `+`**
   - Cause: Mixing `str + int` or `str + float`.
   - Fix: Convert both to the same type first (e.g., `str(25) + str(10)` → `"2510"`; `int("25") + int("10")` → `35`).

5. **`NameError` after `del`**
   - Cause: Referenced a variable after deleting it.
   - Fix: Don't use the variable after `del` — or comment out the `del` line if you need it again.
