# Mini Project 2: Personal Profile Generator

**Duration:** 30–45 minutes  
**Type:** 🛠️ Project  
**Level:** Noob → Nerd  
**Session:** MP2 (curriculum slot S10)

---

## 🎯 Project mission

Build an interactive **profile generator** that collects information, stores **multiple profiles**, and prints a readable summary — using everything from Phase B: loops, debugging, lists, and dictionaries.

### Features (core)

- Collect **name**, **age**, **hobbies** (comma-separated), and **goal**.
- Validate age as a whole number; reject empty name or goal.
- Store hobbies as a **list** inside each profile **dict**.
- Keep **many profiles** in a list; commands: `new`, `list`, `show <n>`, `quit`.

### Stretch goals (optional)

- Export one profile to a `.txt` file with `open()` (preview of file I/O in Level 2).
- Search profiles by name substring.
- Edit an existing profile by number.

---

## 🧰 Troubleshooting

1. **Age validation fails for valid input**  
   `isdigit()` requires whole numbers like `25` — no decimals or spaces.

2. **Empty hobbies list**  
   That's OK — display `(none)`; hobbies are optional.

3. **show command says invalid number**  
   Use `list` first to see valid indices starting at `1`.

---

## ✅ Success criteria

- At least two profiles can be stored and listed in one run.
- Invalid age or empty required fields are rejected without crashing.
- Output is readable without reading the source code.
